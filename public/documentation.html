Movies API

An API for an application called “myFlix” that interacts with a database that stores data about different movies

Documentation of Endpoints

Business Logic URL HTTP Method Request Body Data Format Response Body Data Format
Allows a user tp log-in into their account. /login POST (Create) In order to log-in it's necessary to pass two parameters in the Parameter section: Username and Password in order to get hold of the personalized JWT Token. This token needs to be applied in the Section "Authorization" under JWT Token. This allows the user to use other restricte endpoints. A JSON object holding data about the user: user ID, Username, hashed Password, Email and DateofBirth AND a personalized JWT Token. The uuid creates a new unique ID for the new user.
Allows to get a list of all users /users GET (Read) A list of JSON objects holding data about the Users: user ID, Username, Password, Email and DateofBirth.
Allows to get a User by Username /users/:Username GET (Read) No data in body A JSON object holding data about the user: userID, Username, Password, Email and DateofBirth.
Allows users to register /users POST (Create) A JSON object holding data about the user: Username, Password, Email and DateofBirth. A JSON object holding data about the user: user ID, Username, Password, Email and DateofBirth. The uuid creates a new unique ID for the new user.
Allows to update their username /users/:Username PUT (Update) A JSON object holding data about the user: NEW Username, Password, Email and DateofBirth. A JSON object holding data about the user: user ID, updated Username, Password, Email and DateofBirth.
Allows the user to add a movie to their favorits with just an info that it has been added /users/:Username/movies/:movieID POST (Create) None. The ID number of the new favorite movie is being coveyed as an attribute. A text message that a movie has been added.
Allows the user to delete a movie from their favorits with just an info that it has been deleted /Users/:Username/Movies/:movieID DELETE None. The ID number of favorite movie to be deleted from the list of favorite movies is being coveyed as an attribute. Just a text message that the movie has been deleted from the favorites
Allows the user to delete their user name /users/:Username DELETE None. The Username to be deleted from the list of all users is being coveyed as an attribute. Just a text message that the username has been deleted
Gets the list of ALL movies and their Data in JSON /movies GET (Read) None A JSON object holding the data about the movies (title, release year, description, genre (name and definition), director (name, bio, year of birth and death, if applicable), image URL, whether it’s featured or not)
Gets the data in JSON of a chosen movie by title. /Movies/:Title GET (Read) None. The movie title is being coveyed as an attribute. A JSON object holding the data about the movies (title, release year, description, genre (name and definition), director (name, bio, year of birth and death, if applicable), image URL, whether it’s featured or not)
Gets the data about a genre (description) by name (e.g., “Thriller”) /movies/genre/:genreName GET (Read) None. The genre name is being coveyed as an attribute. A JSON object holding the data about a genre: name and its description.
Gets the data about a director (bio, birth year, death year) by name; /movies/directors/:directorName GET (Read) None. Director's name is being coveyed as an attribute. A JSON object holding the data about a director: name, bio and the year of birth and death, if applicable.
